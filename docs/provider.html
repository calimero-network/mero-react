MeroProvider

Root context provider — wrap your app once, access auth state anywhere

Basic Setup

import { MeroProvider } from '@calimero-network/mero-react';

function App() {
  return (
    <MeroProvider
      nodeUrl="http://localhost:2428"
    >
      <YourApp />
    </MeroProvider>
  );
}

Place MeroProvider at the root of your component tree. It initializes the MeroJs SDK instance, checks for SSO tokens in the URL hash, restores stored tokens, and manages the token refresh cycle.

MeroProvider Props

nodeUrl: string
Required. Base URL of the Calimero node. Used to initialize the SDK instance.
storageKey?: string
localStorage key for persisting tokens. Default: 'calimero_tokens'.
onAuthRequired?: () => void
Called when no tokens are found and no SSO hash is present. Use to redirect to login.
children: ReactNode
Your application tree.

Initialization Flow

1

Check URL hash

Calls parseAuthCallback(window.location.hash). If tokens found, stores them and clears the hash.

2

Restore stored tokens

Reads tokens from localStorage. If valid and not expired, initializes auth state silently.

3

Auth required

If neither hash nor stored tokens present, calls onAuthRequired(). Provider renders children in unauthenticated state.

4

Token refresh

Schedules automatic refresh before expiry. Tokens are updated in storage transparently.

Accessing the SDK

The MeroJs instance is available via the useCalimero hook:

import { useCalimero } from '@calimero-network/mero-react';

function MyComponent() {
  const { sdk, isAuthenticated, logout } = useCalimero();

  const handleLoad = async () => {
    const contexts = await sdk.admin.getContexts();
    // ...
  };
}
