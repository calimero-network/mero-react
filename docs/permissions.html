Permissions

AppMode controls which permissions are requested during the auth flow

How permissions work

When connectToNode(url) is called, the provider builds an auth URL that includes a permissions list. The node's auth page shows the user exactly what your app is requesting. The mode you pass to MeroProvider determines this list.

import { MeroProvider, AppMode } from '@calimero-network/mero-react';

<MeroProvider mode={AppMode.SingleContext}>
  <App />
</MeroProvider>

AppMode.SingleContext

Permissions requested: context:execute

The app works within one specific context. The node selects and assigns the context during login. Use this for most apps — games, collaborative tools, any app where the context is fixed per install.

context:execute
Call contract methods (RPC) in the assigned context.
// After login, contextId and contextIdentity are set automatically.
// Just read them from useMero() and pass to useExecute().
const { contextId, contextIdentity } = useMero();
const { execute } = useExecute(contextId, contextIdentity);

AppMode.MultiContext

Permissions requested: context:create, context:list, context:execute

The app manages multiple contexts. The user can create new contexts and switch between them. Use this for dashboards, admin tools, or apps where users operate across different groups or projects.

context:create
Create new contexts via useCreateContext().
context:list
List existing contexts via useContexts().
context:execute
Call contract methods in any listed context.
// User picks a context from a list:
const { contexts } = useContexts(applicationId);
const [activeCtx, setActiveCtx] = useState(null);
const { execute } = useExecute(activeCtx?.contextId, contextIdentity);

AppMode.Admin

Permissions requested: admin

Full administrative access to the node. Use for internal tooling, developer dashboards, and management apps. Not intended for end-user applications.

admin
Access all admin API endpoints — install applications, manage contexts, view node health, manage peers.
// With admin mode you can access mero.admin directly:
const { mero } = useMero();
await mero.admin.listApplications();
await mero.admin.installApplication(req);
await mero.admin.healthCheck();

Choosing a mode

Building an app installed via Desktop?
Use SingleContext. Desktop provides the context — your app just executes within it.
Building a multi-tenant or collaborative tool?
Use MultiContext. Let users create and switch between contexts.
Building a node management dashboard?
Use Admin. Gives full API access.

The mode is set once at the provider level. If your app needs to change mode, re-mount the provider with a different value and trigger a fresh connectToNode() call so the node issues new tokens with the updated permissions.
