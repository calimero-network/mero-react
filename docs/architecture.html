Architecture

Provider + Context + Hooks — thin React layer over mero-js

Source Structure

src/index.ts
Public API. Exports MeroProvider, AppMode, useMero, useExecute, useSubscription, and storage helpers.
src/context/
React Context definition and MeroProvider component. Holds the MeroJs instance and auth state. Reads SSO tokens from URL hash on mount.
src/hooks/index.ts
useMero() — reads auth state from context. useExecute() — wraps sdk.rpc.execute() with React state. useSubscription() — sets up and tears down event subscriptions.
src/storage/
Token persistence helpers. Default: localStorage with known key names. Pluggable via MeroProvider prop.
src/types.ts
Shared TypeScript types exposed from the package. Mirrors and re-exports key types from mero-js.

MeroProvider Internals

On mount, MeroProvider does the following:

1

Check URL Hash

Calls parseAuthCallback(window.location.hash) from mero-js. If tokens are present (SSO from Desktop), initialises immediately without showing auth UI.

2

Check Stored Tokens

Checks the TokenStore (default: localStorage) for existing tokens. If found and not expired, restores the session.

3

Auth Required

If no valid tokens are found, sets isAuthenticated = false. The app can render its own login UI and call connectToNode().

4

Token Refresh

Schedules background token refresh before expiry. No manual refresh required from application code.

AppMode

Controls how the provider scopes context selection:

enum AppMode {
  // One context per node — simplest case for single-context apps
  SingleContext = 'single_context',

  // Multiple contexts — user picks which context to operate in
  MultiContext = 'multi_context',
}

Most applications use AppMode.SingleContext. Use MultiContext if your app needs to let users switch between contexts at runtime.

Relationship to mero-js

mero-react is a thin React wrapper. It does not duplicate any SDK logic — it only manages the React lifecycle:

  • Creates the MeroJs instance once and provides it via context
  • Converts imperative SDK calls into reactive hook values
  • Handles cleanup (unsubscribing, aborting requests) in useEffect teardowns
  • Exposes a stable hook API so application components never import from mero-js directly
