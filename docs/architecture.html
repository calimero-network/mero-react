Provider Lifecycle

MeroProvider owns the MeroJs SDK instance and the React context. On mount it runs the following initialisation sequence:

1

Read localStorage

Reads nodeUrl, contextId, contextIdentity, applicationId, and saved tokens from localStorage.

2

Check URL hash

If window.location.hash contains auth tokens (SSO flow), reads them with parseAuthCallback() and applies them via sdk.setTokenData(). Clears the hash from the URL bar.

3

Create MeroJs

Creates a MeroJs instance with the node URL and tokens. Sets isLoading: false when ready. Exposes the instance via context so all hooks can access it.

Context → Hooks

All hooks call useMero() internally to get the SDK instance. This means:

Single source of truth
The SDK instance lives in context — no prop drilling, no duplicated state.
Consistent auth state
All hooks see the same isAuthenticated flag. When tokens expire and refresh, all components re-render consistently.
Null safety
Hooks that call the SDK return null / false / empty state when mero === null (before connection). No crashes.
Stable references
Hook functions are wrapped with useCallback and tracked with useRef — safe to use in dependency arrays.

SSO Flow

// 1. Calimero Desktop opens your app URL with tokens in hash:
// https://your-app.com/#access_token=eyJ...&refresh_token=eyJ...
// &node_url=http://node:2428&application_id=abc&context_id=xyz&context_identity=pub

// 2. MeroProvider.useEffect on mount:
const auth = parseAuthCallback(window.location.hash);
if (auth) {
  sdk.setTokenData({ access_token: auth.accessToken, ... });
  setNodeUrl(auth.nodeUrl);
  setContextId(auth.contextId);
  setContextIdentity(auth.contextIdentity);
  history.replaceState(null, '', location.pathname);
}

// 3. Component code just reads:
const { isAuthenticated, contextId } = useMero();
// isAuthenticated is already true — no login screen needed

Mutation Hooks Pattern

Mutation hooks (create, delete, join, etc.) all follow the same pattern:

const { createContext, loading, error } = useCreateContext();

// Call the mutation:
await createContext({
  applicationId: 'app-id',
  initParams: JSON.stringify({ key: 'value' }),
});

// loading: true while in-flight
// error: Error | null on failure
// The function returns the result or null on failure

The mutation is automatically guarded: if mero === null, sets error to "Not connected" and returns early.

Re-exports from mero-js

mero-react re-exports everything from mero-js, so your app only needs one import:

// Instead of importing from both packages:
import { HTTPError } from '@calimero-network/mero-js';
import { useMero } from '@calimero-network/mero-react';

// You can import everything from mero-react:
import { useMero, HTTPError, RpcError, parseAuthCallback }
  from '@calimero-network/mero-react';
