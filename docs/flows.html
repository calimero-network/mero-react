Auth Flows

How authentication works end to end — connect, redirect, callback, and logout

Standard Connect Flow

The full sequence when a user connects for the first time:

1

connectToNode(url)

Calling connectToNode(url) from useMero() stores the node URL and redirects the browser to the node's auth page via buildAuthLoginUrl(). The callback URL is the current page (hash stripped).

2

Node auth page

The node serves its own login UI. The user authenticates (username/password or another provider). The node issues tokens and redirects back to your app's callback URL with tokens in the hash.

3

Token callback

On remount, MeroProvider detects the hash via parseAuthCallback(window.location.href), stores the tokens, clears the hash from the URL bar, and sets isAuthenticated = true.

// Minimal connect UI
function ConnectPage() {
  const { isAuthenticated, isLoading, connectToNode } = useMero();

  if (isLoading) return <p>Restoring session...</p>;
  if (isAuthenticated) return <App />;

  return (
    <button onClick={() => connectToNode('http://localhost:4001')}>
      Connect to Node
    </button>
  );
}

Session Restore Flow

On every page load, MeroProvider tries to restore the previous session before rendering children:

// What happens on mount (simplified):
// 1. Read nodeUrl from localStorage
// 2. Read tokens from localStorage (access_token, refresh_token)
// 3. Create MeroJs instance with saved nodeUrl
// 4. Call admin.getContexts() to verify tokens are still valid
// 5a. Valid → isAuthenticated = true, isLoading = false
// 5b. Invalid → isAuthenticated = false, isLoading = false

Use isLoading to gate your UI while this check runs. It completes in one round-trip to the node.

const { isAuthenticated, isLoading } = useMero();

if (isLoading) return <Spinner />;
if (!isAuthenticated) return <ConnectPage />;
return <App />;

Token Lifecycle

access_token
Short-lived JWT. Used as Authorization: Bearer on every request. Expiry parsed from JWT payload.
refresh_token
Long-lived token. Used by the SDK to silently refresh the access token when it expires. Handled by LocalStorageTokenStore inside the provider.
expires_at
Timestamp (ms). Parsed from the JWT exp claim. Falls back to now + 1h if the token has no expiry.

Token refresh happens transparently inside the SDK — your component code never needs to handle it. If a refresh fails (e.g. refresh token expired), the SSE connection emits a 401 error and the provider calls logout() automatically.

Logout Flow

Calling logout() from useMero() does the following synchronously:

// logout() implementation:
// 1. mero.clearToken() — wipes tokens from the SDK instance
// 2. mero.close() — closes SSE connection
// 3. clearAllStorage() — removes all mero:* localStorage keys
// 4. Reset all state — mero=null, isAuthenticated=false, etc.

After logout(), isAuthenticated becomes false immediately, causing any guarded component to re-render to the connect state.

Online / Offline Detection

After authentication, the provider opens an SSE connection to the node. isOnline reflects whether that connection is active — no polling.

isOnline = true
SSE connection active. Node is reachable.
isOnline = false
SSE connection dropped. Node may be unreachable. SDK queues/retries requests.
const { isAuthenticated, isOnline } = useMero();

{isAuthenticated && !isOnline && <Banner>Reconnecting...</Banner>}
