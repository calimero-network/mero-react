Token Storage

How tokens are persisted and restored across sessions

Default Storage

MeroProvider uses localStorage by default. Tokens survive page reloads and new tabs in the same origin:

// Default — tokens stored in localStorage
<MeroProvider nodeUrl="http://localhost:2428">
  <App />
</MeroProvider>

// Custom localStorage key:
<MeroProvider
  nodeUrl="http://localhost:2428"
  storageKey="my_app_tokens"
>
  <App />
</MeroProvider>

Custom TokenStore

Pass a custom TokenStore implementation for non-browser environments (e.g. React Native, Electron, encrypted storage):

import { MeroProvider, TokenStore } from '@calimero-network/mero-react';

const secureStore: TokenStore = {
  get: () => SecureStorage.getItem('tokens'),
  set: (tokens) => SecureStorage.setItem('tokens', tokens),
  clear: () => SecureStorage.removeItem('tokens'),
};

<MeroProvider nodeUrl="..." tokenStore={secureStore}>
  <App />
</MeroProvider>

TokenStore Interface

interface TokenStore {
  get(): StoredTokens | null;
  set(tokens: StoredTokens): void;
  clear(): void;
}

interface StoredTokens {
  accessToken: string;
  refreshToken: string;
  expiresAt: number; // Unix timestamp ms
}

Token Lifecycle

On mount
Provider reads TokenStore.get(). If tokens exist and expiresAt > Date.now(), initializes auth state without network call.
Auto-refresh
Provider schedules a refresh 60 seconds before expiresAt. On success, calls TokenStore.set() with new tokens.
On logout
Calls sdk.auth.logout() (revokes server-side), then TokenStore.clear().
SSO override
If URL hash contains tokens (from Tauri desktop app), those take precedence and overwrite any stored tokens.
