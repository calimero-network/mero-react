SSO Integration

How Calimero Desktop opens your app already authenticated

How it works

When Calimero Desktop opens an installed application, it already holds valid tokens for that node. Instead of making the user log in again, Desktop passes the tokens directly in the URL hash. MeroProvider detects and consumes them automatically on mount.

// Desktop opens your app at:
https://your-app.example.com/
  #access_token=eyJhbGci...
  &refresh_token=eyJhbGci...
  &node_url=http%3A%2F%2Flocalhost%3A4001
  &application_id=abc123
  &context_id=ctx456
  &context_identity=ed25519%3Apubkey...

MeroProvider reads these on mount via parseAuthCallback(window.location.href), stores the tokens, and immediately sets isAuthenticated = true — the user never sees a login screen.

URL hash parameters

access_token
JWT access token. Used as Authorization: Bearer for all API calls.
refresh_token
Long-lived token for silently refreshing the access token.
node_url
The node's base URL (URL-encoded). Stored in localStorage as mero:node_url.
application_id
The application ID for this install. Stored as mero:application_id.
context_id
The active context ID. Stored as mero:context_id. Available via useMero().contextId.
context_identity
The executor public key for this context. Stored as mero:context_identity. Use as executorId in useExecute.

What MeroProvider does with them

// On mount, provider runs (StrictMode-safe via ref):
const auth = parseAuthCallback(window.location.href);
if (auth) {
  // 1. Store tokens
  tokenStore.setTokens({ access_token, refresh_token, expires_at });
  // 2. Persist all values to localStorage
  setNodeUrl(auth.nodeUrl);
  setApplicationId(auth.applicationId);
  setContextId(auth.contextId);
  setContextIdentity(auth.contextIdentity);
  // 3. Clean up URL (keep path + query, drop hash)
  history.replaceState(null, '', location.pathname + location.search);
}

Writing SSO-compatible apps

Because MeroProvider handles the SSO tokens automatically, you don't need any special code. The same app works both ways:

Opened from Desktop

Tokens arrive in the hash → isAuthenticated = true immediately. User skips login.

Opened standalone

No hash tokens → isAuthenticated = false. User sees your connect UI and logs in normally.

// This component works in both cases — no changes needed:
function App() {
  const { isAuthenticated, isLoading, connectToNode } = useMero();

  if (isLoading) return <Spinner />;
  if (!isAuthenticated) return <ConnectButton />;
  return <Dashboard />;
}

After SSO — using contextId and contextIdentity

After a successful SSO open, contextId and contextIdentity are already populated — you can make RPC calls immediately:

function Dashboard() {
  const { contextId, contextIdentity } = useMero();
  const { execute } = useExecute(contextId, contextIdentity);

  // Ready to go — no extra setup needed
  const data = await execute<MyData>('get_data');
}
