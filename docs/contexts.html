Contexts

A context is a running instance of a WASM application on the node

What is a context?

A context ties together an application (WASM binary), a group of participants, and an execution environment. Every RPC call targets a specific context ID. Each context has:

contextId
Unique identifier for the running application instance.
applicationId
The application (WASM bundle) installed in this context.
contextIdentity
The executor's public key — your identity within this context. Used as executorId for RPC calls.

In AppMode.SingleContext, MeroProvider reads contextId and contextIdentity from the auth callback and stores them in localStorage. Access them via useMero().

Reading the active context

const {
  contextId, // string | null — active context ID
  contextIdentity, // string | null — your executor public key
  applicationId, // string | null — the installed application
} = useMero();

// Pass to useExecute to make RPC calls:
const { execute } = useExecute(contextId, contextIdentity);

Listing contexts

import { useContexts } from '@calimero-network/mero-react';

function ContextList({ applicationId }) {
  const { contexts, loading, error, refetch } = useContexts(applicationId);

  // contexts: Array<{ contextId: string; applicationId: string }>
  return (
    <ul>{contexts.map(c => <li key={c.contextId}>{c.contextId}</li>)}</ul>
  );
}

Pass applicationId to filter by application, or omit it to list all contexts on the node. useApplicationContexts is an alias for the same hook.

Creating a context

import { useCreateContext } from '@calimero-network/mero-react';

function CreateContext({ applicationId }) {
  const { createContext, loading, error } = useCreateContext();

  const handle = async () => {
    const result = await createContext({
      applicationId,
      initParams: JSON.stringify({ key: 'value' }), // optional init args
    });
    console.log('created:', result?.contextId);
  };

  return <button onClick={handle} disabled={loading}>New Context</button>;
}

Joining an existing context

import { useJoinContext } from '@calimero-network/mero-react';

const { joinContext, loading, error } = useJoinContext();

await joinContext({ contextId: 'ctx-123', invitationPayload: payload });

Context discovery

useContextDiscovery polls the node until a context matching your application is found — useful for apps where another peer creates the context and you need to join it.

import { useContextDiscovery } from '@calimero-network/mero-react';

const { context, loading, error, discover, reset } = useContextDiscovery({
  applicationId: 'my-app-id',
  knownContextIds: ['already-joined-ctx'], // skip these
  pollIntervalMs: 2000,
  timeoutMs: 30000,
});

// context: ApplicationContextRecord | null when found
// call discover() to start, reset() to clear result

Other context hooks

useDeleteContext()
Returns { deleteContext, loading, error }. Permanently removes a context from the node.
useContextGroup(contextId?)
Fetches the group that a context belongs to. Returns { data, loading, error }.
useApplicationContexts(applicationId?)
Alias for useContexts — returns the same { contexts, loading, error, refetch }.
